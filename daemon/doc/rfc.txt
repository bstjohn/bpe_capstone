		<pre id="file_contents">The Bonneville Power Administration web application

Abstract
    
  This document defines communication between the web front end
  and the query application for the Bonneville Power Administration 
  web application.

  The term &quot;client&quot; will be used for the web front-end. The
  term &quot;server&quot; will be used for the back-end query application.

  All messages will be in the JSON format.

  There are four types of messages.
    
  Query Request, Query Response, Status Request, Status Response

  &quot;Query Request&quot; and &quot;Status Request&quot; are sent by the client, and
  &quot;Query Response&quot; and &quot;Status Response&quot; are sent by the server.

  The messages are defined below.

Connection

  The server will listen on port 1701 and wait for a TCP 
  connection from client. Connection should only be accepted from 
  a specified IP address. IP address should be exchanged during 
  application set up.

  If the connection is broken, the client will try to reconnect
  every 30 seconds. 

Messages

  Message will be sent using the JSON format specified http://www.json.org/
  
  http://www.json.org/ should be consulted for JSON message details such 
  as escape codes, whitespace, etc...

  All dates will be in the POSIX format.

Query Request

  This message is used by the client to request the server run a query, 
  and return the results. For details about this message&#039;s variables, consult 
  &quot;PMU Identification Numbers Technical Notes&quot; pages 3-7.

        {
            &quot;query&quot;: {
                &quot;query_id&quot;: number,
                &quot;created&quot;:  &quot;string&quot;,
                &quot;start&quot;:    &quot;string&quot;,
                &quot;end&quot;:      &quot;string&quot;,
                &quot;stations&quot;: [&quot;string&quot;],
                &quot;conditions&quot;: {
                    &quot;voltage&quot;: [&quot;string&quot;],
                    &quot;current&quot;: [&quot;string&quot;],
                    &quot;freq&quot;:    [&quot;string&quot;]
                },
                &quot;signal&quot;: {
                    &quot;measurement&quot;:  &quot;string&quot;,
                    &quot;nomvolts&quot;:  number,
                    &quot;circuit&quot;:  number,
                    &quot;identifier&quot;:  &quot;string&quot;,
                    &quot;suffix&quot;:  &quot;string&quot;
                },
                &quot;user&quot;: {
                    &quot;name&quot;: &quot;string&quot;
                },
                &quot;analysis&quot;: {
                    &quot;file&quot;:  &quot;string&quot;,
                    &quot;type&quot;:  &quot;string&quot;,
                    &quot;content&quot;:  &quot;string&quot;
                }
            }
        }

        query_id:     A unique ID given to every query
        created:      Time request was created
        start:        Beginning query period time
        end:          End of query period time
        stations:     List of stations 
        voltage:      List of voltages constraints
        current:      List of currents constraints
        freq:         List of frequencies constraints
        measurement:  One character to identify measurement
        nomvolts:     Nominal Volts
        circuit:      Circuit number
        identifier:   Measurement identifier
        suffix:       Suffix (Defined on page 6 of PMU notes)
        name:         Name of user making request
        file:         Name of analysis file
        type:         File type/suffix
        content:      Content of analysis file

  Example:

        {
            &quot;query&quot;: {
                &quot;query_id&quot;:  321,
                    &quot;created&quot;:  &quot;2014-11-15 16:00:00&quot;,
                    &quot;start&quot;:  &quot;2014-11-15 16:00:45&quot;,
                    &quot;end&quot;:  &quot;2014-11-15 16:01:00&quot;,
                    &quot;stations&quot;:  [&quot;Albon&quot;, &quot;Anne&quot;],
                    &quot;conditions&quot;: {
                        &quot;voltage&quot;: [&quot;voltage &gt; 500&quot;],
                        &quot;current&quot;: [&quot;current &lt; 500&quot;],
                        &quot;freq&quot;: [&quot;freq = 60&quot;]
                    },
                    &quot;signal&quot;: {
                        &quot;measurement&quot;: &quot;B&quot;,
                        &quot;nomvolts&quot;: 3,
                        &quot;circuit&quot;: 2,
                        &quot;identifier&quot;: &quot;VP&quot;,
                        &quot;suffix&quot;: &quot;M for magnitude&quot;
                    },
                    &quot;user&quot;: {
                        &quot;name&quot;: &quot;Sam I Am&quot;
                    },
                    &quot;analysis&quot;: {
                        &quot;file&quot;: &quot;foobar&quot;,
                        &quot;type&quot;: &quot;r&quot;,
                        &quot;content&quot;: &quot;print ( &quot;Hello World!&quot;)\n&quot;
                    }
            }
        }


Query Response

  This message should be returned by the server after receiving a Query request. 
  Status code is a non-negative integer, 0 being success.

        { 
            &quot;QueryResponse&quot;: {
                &quot;statusCode&quot;: number,
                &quot;message&quot;:&quot;String&quot;
            }
        }

  Example of success:

        { 
            &quot;QueryResponse&quot;: {
                &quot;statusCode&quot;: 0,
                &quot;message&quot;:&quot;&quot;
            }
        }

  Error example:

        { 
            &quot;QueryResponse&quot;: {
                &quot;statusCode&quot;: 1,
                &quot;message&quot;:&quot;Malformed query&quot;
            }
        }
  

Status Request

  This message is used by the client to request the status of a queries. There
  are three possible types of request: &quot;Status&quot;, &quot;Results&quot;, and &quot;Analysis&quot;.

        { 
            &quot;StatusRequest&quot;: [&quot;String&quot;] 
        }

  Example:
      
        { 
            &quot;StatusRequest&quot;: [&quot;Status&quot;, &quot;Results&quot;]
        }


Status Response

  This message is used by the server to respond to a status request 
  message. &quot;QueryResponses&quot; is an array of objects that contains the results
  of a search. &quot;AnalysisResponses&quot; is an array of objects that contains
  the results of running the analysis program on the query results. 
  &quot;StatusResponses&quot; is an array of objects that contains the status
  of a running query. The content of &quot;QueryResponse&quot; is the result of a 
  completed query. The content of a &quot;AnalysisResponse&quot; is the result of a 
  completed analysis program.

        { &quot;Status&quot;: 
            {
                &quot;QueryResponses&quot;: 
                    [ {
                        &quot;query_id&quot;: number,
                            &quot;file&quot;: &quot;string&quot;,
                            &quot;type&quot;: &quot;string&quot;,
                            &quot;content&quot;: &quot;string&quot;
                    } ],

                    &quot;AnalysisResponses&quot;: 
                        [ {
                            &quot;query_id&quot;: number,
                            &quot;file&quot;: &quot;string&quot;,
                            &quot;type&quot;: &quot;string&quot;,
                            &quot;content&quot;: &quot;string&quot;
                        } ],

                    &quot;StatusResponses&quot;: 
                        [ { 
                            &quot;query_id&quot;: number,
                            &quot;CPU&quot;: [number],
                            &quot;completed&quot;: number
                        } ]
            }
        }

        query_id:     A unique ID given to every query
        file:         Name of response
        type:         File type/suffix
        content:      Content of response
        CPU:          List of CPU usage percentages
        completed:    Estimated percentage completed

  Example:

        { &quot;Status&quot;: 
            {
                &quot;QueryResponses&quot;: 
                    [ {
                        &quot;query_id&quot;: 3,
                            &quot;file&quot;: &quot;foo&quot;,
                            &quot;type&quot;: &quot;csv&quot;,
                            &quot;content&quot;: &quot;col1,col2,\\n1,2\\n3,4&quot;
                    } ],

                    &quot;AnalysisResponses&quot;: 
                        [ {
